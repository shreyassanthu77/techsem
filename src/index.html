Server

A server is a piece of computer hardware or software (computer program) that provides functionality for other programs or devices

  • The devices thet depend on server are called clients
  • and this archetecture is called Client-Server Model
  • servers can provide various functionality such as
    • Data Sharing among multiple clients
    • performing Computation for a client
  • A single server can serve multiple clients,
    and a single client can use multiple servers.
  • Typical servers are database servers, web servers, game servers, and application servers.

Drawbacks of traditional Servers

  1. It is resource intensive as it must run all the time
  2. Not easy to scale

Serverless

It is an on demand cloud computing execution model
cloud provider allocates machine resources on demand, taking care of the servers on behalf of their customers

When the server is not in use there are no resources allocated to the program

  • Pricing is based on the actual amount of resources consumed by an application.
  • "Serverless" is a misnomer in the sense that servers are still used However developers of serverless applications are not concerned with capacity planning, configuration, management, maintenance, operating or scaling of Physical servers

Examples of some serverless platforms

  • Google - Firebase
  • Amazon - AWS Lambda
  • Microsoft - Azure

Advantages

  • Cost Effective
  • Highly Scalable
  • Very Secure
  • Very Little to No Maintainance

Disadvantages

  • Slow Cold Starts
  • Vendor Lock-in

Conclusion

Serverless computing is an on-demand cost effective server platform to build services, run databases with very little to no maintainance but must be beware of vendor lock in and slow cold starts
